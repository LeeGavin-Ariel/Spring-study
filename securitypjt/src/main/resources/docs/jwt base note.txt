# Web은 TCP 통신을 한다.

<CIA>
C : confidentiality 기밀성 (문서의 탈취를 막는다)
I : integrity 무결성 (문서의 변형을 막는다)
A : availability 가용성 (문서의 사용 가능)

-> 문서의 암호화 -> 탈취를 하더라도 변형과 사용을 막을 수 있음

# 두 문제를 해결해야 보안 문제를 해결할 수 있다. 

<RSA>
public key : 공개키
private key : 개인키

송신자는 수신자의 공개키로 암호화하고 수신자는 개인키로 복호화 
수신자의 공개키 -> 수신자의 개인키 (암호화)

송신자가 자신의 개인키로 암호화하고 수신자는 송신자의 공개키로 복호화
송신자 개인키 -> 송신자 공개키 (전자서명 - 송신자 구분이 가능) -> 인증

# 수신자의 공개키로 암호화 -> 송신자의 개인키로 암호화 한다면
인증, 암호화 문제가 모두 해결가능하다. 

<RFC 7519>

RFC 문서 : 약속된 규칙이 정의된 문서(약속된 문서의 방식을 프로토콜이라고 한다)
모든 약속의 프로토콜이 합쳐져서 HTTP 프로토콜이다.
